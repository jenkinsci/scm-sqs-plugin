Konfiguriert Jenkins dafür eine Amazon SQS Queue auf Nachrichten zu überwachen. Jenkins wird ein neues Build starten, wenn eine Nachricht in der Queue abgelegt wurde und der Inhalt der Nachricht mit dem Repository und dem Branch dieses Jobs übereinstimmt. Das Repository sollte so konfiguriert sein, dass es bei Änderungen Nachrichten in der Queue ablegt.

Verfügbare Queues können in der System Konfiguration von Jenkins konfiguriert werden.

Weiterführende Dokumentation ist im SCM SQS Plugin Repository auf GitHub verfügbar.
